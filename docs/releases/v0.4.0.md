# llmTools v0.4.0

`v0.4.0` adds a dedicated, local-only meeting transcription workflow and packages the Phase 4/4.x media, language-routing, and speaker-processing improvements into one release.

## 功能

- 会议转写与纪要：独立原生窗口，支持麦克风、系统音频和本地音频/视频文件。
- 能力自适应 ASR：VibeVoice 一类模型可联合输出文字、说话人和时间戳；普通实时 ASR 先输出文字，再异步回填本地说话人标签；分离不可用时继续仅转写。
- 可编辑会议结果：支持修改完成的转写文字、重命名说话人、合并 speaker，并分别取消最终整理或纪要生成。
- 本地中文纪要：停止后使用本地 GGUF/MLX 文本模型生成摘要、关键决策、待办事项、开放问题和主题。
- 本地恢复与导出：异常退出后可恢复不含临时音频的会议草稿；可导出 Markdown、TXT 和 JSON。
- 媒体与实时字幕：继续支持本地音视频转写、字幕翻译、SRT/VTT/TXT/Markdown 导出，以及麦克风/系统音频实时字幕。
- 本地语言与说话人处理：包含 fastText 语言识别、fast MT 路由、文件说话人分离和网页翻译 cache v2 引擎隔离。
- 窗口置顶：Quick Action、选择操作、悬浮组件、实时字幕和会议窗口均可通过图钉按钮保持置顶。
- 任务取消与长文本处理：取消媒体字幕或会议收尾会终止底层 ASR/pyannote 子进程；长字幕翻译会按模型上下文拆批，超长单条字幕继续按文本边界切分。
- mlx-community VibeVoice-ASR：作为 file-only 原生说话人模型复用 MLX ASR runtime 和本地 Qwen2.5 tokenizer sidecar；原始 PyTorch runtime 仍可通过 legacy 安装脚本或自定义命令使用。
- 有界本地音频工作区：原生说话人 ASR 的连续讲话按 120 秒技术窗口处理；若本地 ASR 慢于采集并排队 2 个窗口，app 会自动停止采集并完成队列；临时目录和恢复草稿使用 owner-only 权限，失败、取消、正常停止或崩溃恢复后会按所有权清理。

## 使用方法

1. 从本 Release 下载 `llmTools-v0.4.0-macos-arm64.zip`，解压并把 `llmTools.app` 移到 `/Applications` 或其他可信目录。
2. 首次启动时按需授予 Accessibility 和麦克风权限；系统音频会议还需要在“系统设置 -> 隐私与安全性 -> 屏幕录制”允许 llmTools，并在授权后重启 app。该构建使用 ad-hoc 签名且未 notarize，必要时从 Finder 的“打开”或“系统设置 -> 隐私与安全性”允许启动。
3. 在“设置 -> 模型”注册并启用需要的本地模型；媒体/会议 ASR 必须使用本地 speech-capable 模型。
4. 在“设置 -> 会议”选择会议采集 ASR、文件 ASR、可选的本地纪要模型、默认输入和源语言，并运行健康检查。
5. 从状态栏菜单打开当前标题为“会议转写与纪要”的窗口。实时会议选择麦克风或系统音频后开始；已有录音选择本地音频/视频文件。
6. 修改转写或 speaker 后停止会话；按需执行“最终整理”和“生成纪要”，再用“导出”写入 Markdown、TXT 或 JSON。
7. 网页翻译仍使用开发通道：在“设置 -> 网页翻译”修复 bridge，并在 `chrome://extensions` 或 `edge://extensions` 重新加载本 Release 附带的 unpacked extension。

## 说明

- 会议 ASR、说话人分离和纪要生成不会回退到远程 provider。
- `v0.4.0` 的会议采集不支持麦克风与系统音频混合；现有实时字幕仍可使用混合来源。
- 正常停止时默认删除临时会议音频；恢复草稿不保留音频，异常退出后的清理会保留仍由其他活跃 app 进程使用的目录。
- Chromium 扩展仍为手动加载的开发版本，不是 Chrome Web Store 分发版本。
