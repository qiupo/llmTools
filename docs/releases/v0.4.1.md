# llmTools v0.4.1

`v0.4.1` focuses on local speech pipelines, process cleanup, Quick Action reliability, and security boundaries around local credentials, provider traffic, and remote OCR images.

## 功能与修复

- FunASR 组合说话人管线：为官方 Fun-ASR-Nano `model.pt` 增加 Nano + FSMN-VAD + CAM++ + CT-Punc 离线管线，可用于文件字幕和离线会议；没有有效说话人标签时继续回退到 pyannote 或仅转写。
- 语音模型设置：按模型族、实时/文件用途、说话人能力和实际 runtime 展示已注册语音模型，并支持安装、验证或修复 FunASR 组合管线。
- 进程与内存回收：会议停止、取消和全局空闲卸载会回收 ASR、Fast MT 与 fastText LID sidecar；阻塞中的请求可被非阻塞 stop 中断，避免结束后模型进程继续占用内存。
- Quick Action 稳定性：切换输入或关闭面板会取消旧任务；划词捕获优先使用 Accessibility，替换原文前校验原选区，剪贴板 fallback 仅在仍持有写入所有权时恢复内容。
- OCR 输入边界：远程图片仅允许公开 HTTPS 地址，并检查重定向、响应类型、下载字节数、输入文件大小和像素上限；取消操作会终止正在进行的图片下载与预处理。
- Provider 与本地 bridge 安全：远程 Provider 要求 HTTPS，本机 loopback 可继续使用 HTTP；跨 origin 或降级重定向被拒绝；本地 bridge 对 HTTP framing 和 body 大小执行有界解析。
- 本地数据权限：继续使用 app 管理的本地 Provider key，不接入 macOS Keychain；Application Support 目录收紧为 `0700`，registry、history 和 backup 文件收紧为 `0600`。
- 文本任务默认模型：翻译、润色、总结、解释和待办可分别选择默认文本模型，未单独配置时回退到 Text 默认模型。
- 运行时健壮性：统一并发读取子进程 stdout/stderr，避免大输出导致 pipe backpressure 卡住；ASR runtime 路径集中到 app 管理目录。

## 使用方法

1. 从本 Release 下载 `llmTools-v0.4.1-macos-arm64.zip`，解压并把 `llmTools.app` 移到 `/Applications` 或其他可信目录。
2. 首次启动时按需授予 Accessibility、麦克风和屏幕录制权限；该构建使用 ad-hoc 签名且未 notarize，必要时从 Finder 的“打开”或“系统设置 -> 隐私与安全性”允许启动。
3. 在“设置 -> 模型”注册并启用本地/远程模型。远程 Provider 使用 HTTPS；仅 `localhost` 和 loopback 地址允许 HTTP。
4. 在“设置 -> 文本”选择默认模型，并按需为翻译、润色、总结、解释和待办分别指定模型。
5. 在“设置 -> 媒体字幕”查看语音模型能力。使用官方 Fun-ASR-Nano 时，可从设置中安装并验证离线组合管线。
6. 网页翻译扩展仍为开发通道；升级后请在 `chrome://extensions` 或 `edge://extensions` 重新加载本 Release 附带的 unpacked extension。

## 说明

- ASR、说话人处理和会议纪要继续保持 local-only，不会回退到远程 Provider。
- 官方 FunASR 组合管线用于文件字幕和离线会议；根据当前 speaker 质量测试，它不进入实时会议采集模型选择。
- Provider API key 仍保存在本地 `model-registry.json`。权限已收紧，但共享 macOS 账户或可读取用户文件的备份仍属于同一信任边界。
- Chromium 扩展仍为手动加载的开发版本，不是 Chrome Web Store 分发版本。
- macOS app 使用 ad-hoc 签名且未 notarize。
